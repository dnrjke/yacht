# P2 구현 이후 추가 작업 목록

> 작성일: 2026-04-07
> 전제: `p2-issue-audit.md` Phase A~D 완수 후 실제 2인 온라인 플레이를 위해 필요한 작업

---

## 1. 필수 (없으면 플레이 불가)

### 1.1 프론트엔드 로비 UI

- **파일**: `frontend/src/components/screens/MainMenuScreen.tsx`
- **현상**: "멀티플레이" 버튼만 존재, 실제 `JOIN_ROOM` 연결 흐름 없음
- **작업**: 방 생성/참가 화면 구현. 방 ID 입력 또는 자동 매칭 선택지 제공

### 1.2 P2 대기 화면

- **작업**: P1 입장 후 P2가 들어올 때까지 대기 UI 표시
- **트리거**: `ROOM_JOINED` 수신 후 ~ `GAME_START` 수신 전까지
- **표시 내용**: 방 ID (공유용), "상대방을 기다리는 중..." 메시지

### 1.3 상대방 컵 렌더링

- **파일**: `frontend/src/components/3d/PhysicsCup.tsx`
- **현상**: `CUP_TRANSFORM` 릴레이는 구현 가이드에 포함되었지만, PhysicsCup이 서버로부터 수신한 상대방 컵 위치 데이터를 반영하는 로직이 없음 (현재 자기 드래그만 처리)
- **작업**: 내 턴이 아닐 때 수신된 `CUP_TRANSFORM` 데이터로 컵 위치/회전을 갱신하는 로직 추가

### 1.4 GAME_OVER 화면

- **파일**: `frontend/src/components/screens/` (신규)
- **현상**: `GamePhase`에 `'GAME_OVER'`가 정의되어 있으나 화면 미구현
- **작업**: 13라운드 종료 후 양쪽 최종 점수, 승패 표시, "다시 하기" 버튼

---

## 2. 높은 우선순위 (없으면 UX 파탄)

### 2.1 상대 턴 시각 피드백

- **작업**: 내 턴이 아닐 때 명확한 비활성 표시
  - 스코어보드 헤더에 "상대방 차례" 강조
  - 컵/주사위에 비활성 스타일 또는 반투명 오버레이
  - 턴 전환 시 간단한 전환 애니메이션

### 2.2 턴 타이머

- **작업**: 서버에서 턴당 제한 시간 관리 (예: 60초)
- **시간 초과 시**: 서버가 강제로 가장 낮은 점수 카테고리에 기록하거나, 빈 카테고리에 0점 기록
- **클라이언트**: 남은 시간 카운트다운 UI 표시

### 2.3 disconnect 알림

- **작업**: `PLAYER_LEFT` 수신 시 "상대방이 나갔습니다" 팝업
- **선택지**: 게임 종료 (메인 메뉴 복귀) 또는 재접속 대기

---

## 3. 배포 관련

### 3.1 소켓 서버 URL 동적화

- **파일**: `frontend/src/App.tsx`
- **현상**: `io('http://localhost:3001')` 하드코딩
- **작업**: 환경변수 (`VITE_SOCKET_URL`) 또는 빌드 타임 설정으로 전환

### 3.2 배포

- **Backend**: Koyeb (또는 유사 서비스). `implementation_plan.md` Phase 6에 계획됨
- **Frontend**: Vercel. Vite 빌드 후 정적 배포
- **주의**: CORS 설정, WebSocket 프록시, HTTPS 등 프로덕션 설정 필요

---

## 4. 있으면 좋음 (후속 개선)

### 4.1 재접속 (Reconnect)

- 브라우저 새로고침 시 기존 Room에 복귀
- 서버에서 socket.id 변경 대응 (세션 토큰 또는 playerId 쿠키)

### 4.2 관전 모드

- 3번째 이상 접속자를 읽기 전용 관전자로 수용
- `DICE_STATES`, `POUR_RESULT` 등 브로드캐스트 수신만 가능

### 4.3 사운드

- `frontend/public/` 에 에셋 자리 이미 존재
- 주사위 충돌, 컵 흔들기, 점수 기록, 콤보 달성 등 효과음 추가

### 4.4 비주얼 폴리시

- 주사위/컵 3D 모델 교체 (현재 임시 지오메트리)
- 나무 테이블 텍스처, 가죽 컵 재질 등

---

## 구현 권장 순서

Phase A~D 완수 직후, 같은 세션 또는 바로 다음 세션에서:

```
1단계: 1.1 + 1.2 (로비/대기 UI)  ← 연결 흐름 완성
2단계: 1.3 (상대 컵 렌더링)       ← 상대방 행동이 보임
3단계: 1.4 (GAME_OVER 화면)       ← 게임 루프 완결
4단계: 2.1 + 2.3 (턴 피드백 + disconnect)  ← 최소 UX
──── 여기까지 하면 로컬 네트워크에서 2인 플레이 가능 ────
5단계: 3.1 + 3.2 (배포)           ← 인터넷 플레이
6단계: 2.2 (턴 타이머)            ← 악용 방지
7단계: 4.x (후속 개선)            ← 폴리시
```
