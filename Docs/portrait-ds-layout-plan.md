# 모바일 세로 모드 DS형 레이아웃 계획

## 목표
닌텐도 DS의 상/하 2화면 구조에서 영감을 받은 세로 모드 전용 레이아웃 신설.
- 화면 위/아래에 점수판 / 게임 화면 분할 배치
- 점수판은 좌(상단부~Subtotal·Bonus) / 우(하단부~TOTAL) 2단 분할 — 기존 점수판을 잘라붙인 형태
- 턴 플레이어 표시를 점수판의 게임 화면 쪽 가장자리 중앙에 볼록 튀어나온 탭(DS 힌지 느낌)으로 이동
- 홈/오디오 버튼은 점수판 바깥 좌하단, 점수판 위/아래 전환 버튼 추가

## 발동 조건
- `aspectRatio < 1` (세로 모드)일 때 본 레이아웃 적용
- `aspectRatio >= 1`은 기존 가로 레이아웃 그대로 — **기존 코드 경로 무변경**
- 기존 squarish(≤1.5) 처리는 가로 레이아웃 내부 분기이므로 1.0~1.5 구간은 현행 유지

## 레이아웃 구조

```
점수판 상단 배치 (기본값)              점수판 하단 배치 (전환 시)
┌─────────────────────────┐          ┌─────────────────────────┐
│  ┌─ 좌 테이블 ─┬─ 우 테이블 ─┐  │          │                         │
│  │ Aces~Sixes │ Choice~     │  │          │      게임 화면 (3D)      │
│  │ Subtotal   │ Yacht       │  │          │                         │
│  │ Bonus      │ TOTAL       │  │          │                         │
│  └────────────┴─────────────┘  │          ├──┬──────┬─────────┬──┬──┤
├──┬──────┬─────────┬──┬─────────┤          │🏠│ 🔊  │ MY TURN │▼ │  │ ← 밴드(위로 볼록)
│🏠│ 🔊  │ MY TURN │▲ │         │          ├──┴──────┴─────────┴──┴──┤
└──┴──────┴────╲────┴──┴─────────┘          │  ┌─ 좌 테이블 ─┬─ 우 ─┐  │
│       ╲ 볼록 탭(아래로)         │          │  │            │      │  │
│      게임 화면 (3D)            │          └─────────────────────────┘
└─────────────────────────┘
```

- **밴드**: 점수판의 게임 화면 쪽 가장자리에 붙는 높이 ~44px 스트립.
  좌측 = 홈/오디오 버튼, 중앙 = 턴 표시 볼록 탭, 우측 = 위치 전환 버튼.
- **볼록 탭**: 점수판과 같은 배경(#1a1a1a)·테두리로 이어진 둥근 탭.
  게임 화면 방향으로 돌출. 턴 텍스트는 기존 색 규칙(`p1` #4CAF50 / `p2` #2196F3).
- 점수판이 하단으로 가면 밴드 전체가 점수판 위쪽 가장자리로 이동(탭은 위로 볼록).
  점수판 하단 배치 시 화면 맨 아래엔 돌출 공간이 없으므로 게임 화면 쪽을 향하는 게 자연스러움.

## UX 선정 사항 (사용자 위임 항목)

### 홈/오디오 버튼: 가로 배치 선정
- 세로 모드에서 가장 부족한 자원은 세로 공간. 밴드는 턴 탭 때문에 어차피 생기는
  스트립이므로, 같은 밴드 좌측에 가로로 넣으면 **추가 높이 비용 0**.
- 세로 배치는 점수판 옆이나 게임 화면을 침범해야 해서 적합도 낮음.
- 볼륨 팝오버는 버튼 기준 게임 화면 방향으로 펼침 (점수판을 가리지 않도록).

### 위치 전환 버튼: 밴드 우측, 화살표 아이콘
- 홈/오디오와 반대쪽 끝에 배치 — 오조작 분리.
- 아이콘은 점수판이 이동할 방향을 가리킴: 상단 배치 중엔 ▼, 하단 배치 중엔 ▲.
- 선택값은 `localStorage('yacht_portrait_scoreboard_pos')`에 저장, 재방문 시 유지.

## 점수판 2단 분할

| 좌 테이블 | 우 테이블 |
|---|---|
| 헤더 (P1 / P2) | 헤더 (P1 / P2) |
| Aces ~ Sixes (6행) | Choice, FourOfAKind, FullHouse (3행) |
| Subtotal | SmallStraight, LargeStraight, Yacht (3행) |
| Bonus | TOTAL |

- 행 수 좌 9 / 우 8로 거의 균형. 두 테이블 `flex: 1 1 50%`로 좌우 배치.
- 턴 표시는 탭으로 이동했으므로 테이블 헤더는 P1/P2만.
- 점수 기입(placement 모드 셀 클릭) 동작은 기존과 동일.

## 구현 단계

### 1단계: 점수 클릭 로직 공용화
- `Scoreboard.tsx`의 `handleScoreClick`(점수 기입 + 게임 종료 판정 + 턴 전환)을
  `frontend/src/components/ui/useScoreClick.ts` 훅으로 추출.
- 기존 `Scoreboard.tsx`는 훅 호출로 치환만 — 렌더 부분 무변경.
- 이유: 세로 점수판에 로직을 복제하면 게임 종료 판정이 이중 관리됨.

### 2단계: `PortraitScoreboard.tsx` 신규
- 2단 테이블 렌더. 스타일 토큰(색·폰트 최소값)은 기존 Scoreboard와 동일 계열 사용.
- props: `uiScale` (세로 모드용 산정값).

### 3단계: GameScreen 세로 분기
- `aspectRatio < 1`이면 세로 레이아웃 렌더 (column flex):
  - 점수판 패널: 높이 ~40% (내부는 기존처럼 table flex로 채움)
  - 밴드: 44px (버튼 + 탭 + 전환 버튼)
  - 게임 화면: `flex: 1`
- `scoreboardOnTop` state + localStorage 연동. 전환 시 column 순서만 반전.
- 기존 가로 레이아웃 JSX는 그대로 두고 조건 분기로 감쌈.

### 4단계: 검증
- tsc 통과 확인, 가로 모드 회귀 없음 확인(분기 바깥 무변경이므로 구조적으로 보장).
- 실기기 세로 모드에서 확인 필요: 탭 돌출 겹침, 점수판/게임 비율, 전환 버튼 동작.

## 영향 범위
- `GameScreen.tsx` — 세로 분기 추가 (기존 가로 경로 무변경)
- `Scoreboard.tsx` — handleScoreClick → 훅 치환 (동작 동일)
- `ui/useScoreClick.ts` (신규), `ui/PortraitScoreboard.tsx` (신규)
- 카메라: `ResponsiveCameraManager`가 Canvas 컨테이너 비율 기준으로 이미 동작하므로
  추가 작업 없이 새 영역 비율에 맞춰 자동 조정될 것으로 예상 (실기기 확인 항목)

## 리스크 / 미확정
- 점수판 40% 비율은 추정값 — 폰트 최소값에서 9행이 들어가는지 실측 후 조정 가능
- ~~주사위가 작게 보일 수 있음~~ → **검증 결과 기우로 확인**: ResponsiveCameraManager는
  aspect < 1에서 보드 가로폭을 캔버스 가로폭에 맞추므로, 주사위 픽셀 크기는 캔버스
  가로폭에만 비례. DS 모드 캔버스는 기기 전체 폭을 쓰므로 현행 세로 모드(사이드바가
  가로폭 잠식, 캔버스 ~170px)보다 오히려 주사위가 2배 이상 커짐.
  경계 사례: 태블릿 세로 등 하단 영역이 aspect ≥ 1이면 fit 축이 세로로 전환되나
  데스크톱과 동일 규칙이므로 문제 없음
- 볼록 탭 돌출부가 3D 영역의 터치(컵 드래그)와 겹칠 수 있음 — 탭은 pointer-events 영역을
  탭 자체로 한정
